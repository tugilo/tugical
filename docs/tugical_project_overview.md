## プロジェクト概要

### tugical とは（差別化の一言）

tugical は **「予約を管理するツール」ではありません。**

**「予約の“次の行動”までを設計するツール」**です。

- 予約を取るだけでなく、**組み合わせ・次回・継続が自然に起こる設計**まで含む
- 世にある予約管理ツールとの違い：**LINE 完結** × **提案型予約** × **売上につながる行動設計**が軸
- **なぜ LINE か** … 新しい操作を覚えなくていい。お客さんに説明しなくていい。アプリを入れさせなくていい。＝ **学習コストゼロ**で導入の心理ハードルを下げる。

**設計思想（業種の扱い）**  
tugical は初期から業種別に作り込まない。コアは共通にし、業種ごとの差は初期設定値・表示文言・通知テンプレ・提案の出し方など **「テンプレ／オプション」で後から追加**する設計とする。これによりリリースを止めず、導入後に最適化できる。

## 解決する課題と提供価値

### 現場課題

- 電話・紙台帳での予約管理
- ダブルブッキング／無断キャンセル
- 高機能予約ツールの導入ハードル
- 「売り込み」をスタッフに任せると負荷・ムラが出る

### tugical の提供価値（差別化）

tugical は **業務改善ツールではなく“売上設計ツール”** として価値を出す。

- **最初の1回でラクになること（例）** … 電話を取らなくていい時間が増える。次回予約を「言う」必要がなくなる。メニュー説明を仕組みに任せられる。→ 自分事としてイメージしやすい。
- **LINE だけで完結** … 顧客も店舗も、普段使っている LINE のまま。前と同じ感覚で使える。
- **迷わない・失敗しない** … やることを絞り、スタッフに説明しなくていいレベルでそろえられる。
- **提案型予約** … 状況に応じたメニュー提案（売り込みではなく「選んでもらう」設計）。組み合わせメニューで**自然に単価アップ**につながる行動を設計できる。
- **次につながる仕組み** … 次回予約・定期利用を LINE が提案する設計が可能。スタッフが言いづらいことを**仕組みが代わりに言う**。
- 業種テンプレートで導入しやすく、柔軟な予約条件（性別・設備等）に対応。

※「売上が上がる」と断定するのではなく、**「売上につながる行動を設計できる」** ことを提供する。

## 管理者機能（MVP）

- 店舗／施設／スタッフ管理
- メニュー登録（所要時間／金額）・**組み合わせメニュー**（単価アップにつながる設計が可能）
- 予約一覧・タイムライン表示・承認・変更・キャンセル（スタッフに説明しなくていい操作にそろえている）
- LINE 通知編集・業種テンプレートの選択（美容／整体など）
- 空き時間・仮押さえ（10分）による二重予約防止

---

## 顧客側（LINE 連携）

- **LINE だけで完結** … LIFF 起動で顧客識別。別アプリ・会員登録なしで予約まで完結。
- **迷わない** … メニュー・日時選択はタップ中心。テキスト入力は極力なくし、失敗しにくい流れ。
- 前と同じ感覚で使える LINE 内の操作に収め、スマホでそのまま完了。
- 自動 LINE 通知による予約完了案内。
- 予約履歴／キャンセル対応（開発後期）。
